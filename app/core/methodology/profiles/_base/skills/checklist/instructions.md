## Чек-лист проекта

Для практической части подготовь чек-лист проверки проекта в формате `check-list.yml`.

Требования:
- чек-лист покрывает ключевые требования проекта: технический результат, методологические аспекты,
  этапы выполнения и роли, если проект командный;
- каждый пункт проверяется однозначно: ответ должен быть «выполнено» или «не выполнено»;
- формулировки имеют измеримый индикатор: команда запускается без ошибок, есть минимум N тестов,
  файл создан, поле заполнено, результат содержит конкретный элемент;
- не используй субъективные формулировки вроде «аккуратно», «понятно», «качественно», «удобно»;
- чек-лист напрямую отражает задания проекта и не добавляет критерии, которых нет в формулировке.

Если проект чисто исследовательский или теоретический, чек-лист всё равно должен фиксировать
наблюдаемые артефакты: отчёт, аргументы, источники, выводы и проверяемые ограничения.
