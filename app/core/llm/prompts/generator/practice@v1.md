Сгенерируй Главу 3 "Практика" как список проверяемых заданий.

Верни только JSON, совместимый со схемой `PracticeDraft`:

```json
{
  "tasks": [
    {
      "title": "краткое название",
      "situation": "рабочий контекст задачи",
      "constraints_or_risk": "ограничение, риск или точка выбора",
      "input_data": "сырой вход или ссылка на raw materials",
      "goal": "активная цель с глаголом действия",
      "approach_bullets": ["3-6 действий"],
      "expected_artifact": "конкретный проверяемый артефакт",
      "p2p_criteria": ["3-5 бинарных критериев проверки"]
    }
  ]
}
```

Требования:
- количество задач должно совпасть с `task_plan.tasks_count`;
- первая задача работает с raw evidence из `materials/`, следующие используют артефакт предыдущей задачи;
- не давай готовые решения, заполненные таблицы, итоговые матрицы и другие processed materials во входных данных;
- каждый expected artifact должен иметь путь, критерии должны быть наблюдаемыми на p2p;
- не генерируй бонусы, critic/repair, dataset files и code-example bodies: это следующие задачи G4b/G4c;
- для no-code направлений не проси писать код, скрипты, формулы или деплой.
